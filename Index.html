Conceitos de Git e GitHub

Fluxo do Git/GitHub

Comandos básicos:

  • git init - Inicializar repositório
  • git add . - Adicionar arquivos
  • git commit -m "mensagem" - Fazer commit
  • git push origin main - Enviar para GitHub
  • git pull origin main - Baixar mudanças

Como Resolver Conflitos

  • Identificar arquivos em conflito com git status
  • Editar arquivos removendo marcadores de conflito
  • Fazer git add nos arquivos resolvidos
  • Finalizar com git commit

Boas Práticas

Commits Semânticos

  • feat: nova funcionalidade
  • fix: correção de bug
  • docs: documentação
  • style: formatação

Branches

  • main: produção
  • develop: desenvolvimento
  • feature/: novas funcionalidades
  • hotfix/: correções urgentes

CI/CD

Integração e Deploy Contínuo automatizam o processo de desenvolvimento:

  • Integração Contínua: Testes automáticos a cada commit
  • Deploy Contínuo: Publicação automática em produção
  • GitHub Actions: Pipeline de CI/CD integrado

Deploy

Do Git para Produção:

  • GitHub Pages: Hospedagem gratuita para sites estáticos
  • Netlify: Deploy automático conectado ao repositório
  • Vercel: Plataforma para deploy de projetos web
